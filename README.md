故事神谕（Story Oracle）—— SillyTavern 扩展
一个可拖动的侧边面板，连接到你自定义的 OpenAI 兼容 端点，让你针对正在进行中的
剧情向 LLM 提问。问答在它自己独立的窗口、自己独立的历史里进行——完全不会碰到你的主聊天。
随时可以清空。
安装（手动）
把整个 `story-oracle/` 文件夹复制到你的 SillyTavern 第三方扩展目录，然后刷新浏览器标签页。
较新版 ST（按用户分目录的数据结构）：
`SillyTavern/data/<你的用户名>/extensions/story-oracle/`
（默认用户名是 `default-user`）
较旧版 ST：
`SillyTavern/public/scripts/extensions/third-party/story-oracle/`
刷新后，打开魔棒菜单（消息输入框旁边的 🪄 图标）→ 故事神谕。
设置
点击窗口标题栏里的 ⚙ 齿轮。
直连模式（Direct，默认） —— 完全自包含：
端点 URL：你的基础 URL（例如 `https://your-proxy.com/v1`）。如果你只填了主机名或
`/v1` 形式的基础地址，会自动补上 `/chat/completions`。如果想绕过自动补全，直接粘贴
完整的 `.../chat/completions` URL。
API 密钥、模型（例如 `gpt-4o-mini`，或你的代理所提供的任意模型）。
以明文形式保存在 ST 的 settings.json 中 —— 对个人/自建实例来说没问题。
连接配置文件模式（Connection Profile） —— 选择你已保存的某个 ST 连接配置文件。请求会
通过 ST 自己的后端转发，因此 CORS 无法拦截它。如果直连模式抛出网络/CORS 错误，就用这个。
共用选项：温度、最大 token 数、上下文深度（往里喂多少条最近的消息；`-1` = 整段聊天，
`0` = 不喂）、是否包含角色卡的开关，以及一段可编辑的系统提示词。
诊断模式（Diagnose）
点击标题栏里的 🩺 听诊器图标可切换到诊断模式，用于修复 MVU（MagVarUpdate）的状态变量。
在该模式下，神谕会读取本角色卡的 MVU 规则（来自世界书）、当前的实时变量状态（`stat_data`），
以及最新一条 AI 回复里的 `<UpdateVariable>` 区块，然后对照剧情进行核对，最后给出一份保守的
纠正补丁。当回复中包含可用的补丁时，气泡下方会出现一个 将修复应用到状态 的按钮——它会
通过 MVU 自己的管线应用补丁，并支持一键撤销。
说明
系统提示词在每次发送时都会重新构建，所以随着剧情推进，神谕看到的永远是最新的故事状态。
历史保存在内存中：关闭/重新打开窗口会保留它；刷新页面或点击 🗑 清空按钮则会重新开始。
Enter 发送，Shift+Enter 换行。生成过程中，发送按钮会变成停止按钮。
